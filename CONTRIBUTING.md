# Contributing to Bug Bounty Hunting Methodology 2026

Thank you for considering a contribution! This repo is community-driven and improves with every real-world technique shared.

---

## What We Accept

- New vulnerability testing techniques that worked in 2025/2026
- Better one-liners or automation scripts
- Bug fixes in checklists (incorrect steps, outdated tools)
- New tool additions that are actively maintained
- Real write-up references (public disclosures only)
- Corrections to payload lists

## What We Don't Accept

- Proprietary/private exploit code
- Techniques that violate platform terms of service
- Automated mass-scanning scripts targeting live programs
- Social engineering or phishing guides

---

## How to Contribute

1. Fork the repository
2. Create a branch: `git checkout -b feat/your-technique-name`
3. Make your changes
4. Commit: `git commit -m "add: SSTI bypass for Jinja2 with WAF"`
5. Push: `git push origin feat/your-technique-name`
6. Open a Pull Request

---

## Style Guidelines

- Use fenced code blocks with language tags (` ```bash `, ` ```python `)
- Keep explanations concise — this is a field reference, not a tutorial
- Include "why it works" for non-obvious techniques
- Test all commands before submitting
- No broken links

---

## Issues

Found something wrong? Open an issue with:
- The file and section
- What's wrong
- What the correct version should be

---

**This methodology gets better every time someone shares what worked for them.**
